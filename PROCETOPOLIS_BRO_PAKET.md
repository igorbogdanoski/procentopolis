## 3. Педагошко резиме

### 3.1 Опис на играта

Процентополис е дигитална игра на табла инспирирана од класичниот формат „Монопол", но целосно редизајнирана за наставни цели. Сите механики на играта се директно поврзани со математичкото учење на проценти.

**Тек на играта:**
1. Наставникот создава соба со уникатен код и ја дели на учениците
2. Учениците се приклучуваат преку прелиставачот — без регистрација, само со надимок
3. Наставникот ја стартува играта; играчите фрлаат коцка по редослед
4. При секое слетување на ќелија (имот, данок, шанса, бонус) — играчот добива математичко прашање
5. Точен одговор → успешна трансакција; неточен → губи пари / го пропушта купувањето
6. По завршување (40 минути или лимит на потези) — се прикажуваат резултати и рефлексија

**Специјални механики:**
- **Аукција на знаење (Берза)**: кога играч не одговора точно, прашањето оди на аукција — сите играчи може да понудат и да освојат бодови
- **Заем**: играч со мало салдо може да земе заем — со пресметување на камата
- **Изградба**: напредни играчи можат да градат на имот — со пресметување на зголемен наем

### 3.2 Математичка содржина и усогласеност со наставна програма

Играта покрива следните исходи предвидени со наставната програма по математика за VI и VII одделение:

| Тип на прашање | Математичка содржина | Ниво |
|----------------|----------------------|------|
| **BUY – D1** | Пресметај X% од дадена цена | Основно (D1) |
| **BUY – D2** | Пресметај вкупна цена со провизија (цена + X%) | Средно (D2) |
| **BUY – D3** | Нов наем по зголемување за R% | Напредно (D3) |
| **RENT – D1** | X% од вредноста на имотот | Основно (D1) |
| **RENT – D2** | Наем по изградба (зголемен за 20%) | Средно (D2) |
| **RENT – D3** | Нов наем по скриено зголемување | Напредно (D3) |
| **TAX** | Остаток по одбивање на 10% данок | Средно (D2) |
| **BONUS** | 15% од тековниот капитал | Основно (D1) |
| **LOAN** | Камата (X% од 1500) | Напредно (D3) |
| **BUILD** | Зголемување на наем за случаен % | Напредно (D3) |
| **CHANCE** | Прашање според бројот потези (D1→D2→D3) | Адаптивно |

Прашањата се **контекстуално генерирани** — вредностите во прашањето го одразуваат вистинскиот имот и цена во игрaта (не се случајни апстрактни броеви), со што се постигнува **функционален трансфер на знаење**.

### 3.3 Педагошки пристап и теоретска рамка

**Учење засновано на игра (Game-Based Learning / GBL)**
Играта ги применува принципите на GBL: учениците се мотивирани преку награди, натпревар и соработка; математичкото размислување е вградено во одлуките во играта — не е одвоено вежбање.

**Зонски развиток (Зона на блискиот развој — Виготски)**
Адаптивниот систем на тежина (D1 → D2 → D3) обезбедува постепена градација — секој играч почнува со полесни прашања и напредува кон посложени врз основа на бројот на потези.

**Формативна оцена и метакогниција**
По секоја игра, секој ученик добива персонализирано резиме:
- Колку прашања по тип (купување, наем, данок...) добил/пропуштил
- Кои типови му се послаби (за саморефлексија)
- Наставникот добива агрегиран извештај по ученик и по тип на прашање

**Соработничко учење**
Механиката на аукција (Берза на знаење) ги охрабрува учениците да внимаваат и на прашањата на другите играчи — не само на своите — создавајќи колективна ангажираност.

### 3.5 Безбедност и приватност

- **Нема регистрација**: учениците не внесуваат e-mail, лозинка ни какви лични податоци
- **Анонимна автентикација**: Firebase генерира привремен анонимен ID; по затворање на прелиставачот сесијата не постои
- **Без трети страни**: нема реклами, нема трекинг, нема комерцијален модел
- **Нема зачувување на историја**: по завршување на играта соодветните Firebase записи се архивираат/бришат
- **GDPR компатибилност**: нема обработка на лични податоци во смисла на GDPR / Закон за заштита на лични податоци
- **Отворен код**: достапен за инспекција на GitHub (github.com/igorbogdanoski/procentopolis)

## 4. Пристап до апликацијата

### Директна употреба (без инсталација)

Апликацијата е достапна веднаш преку прелиставач:

**https://procentopolis.vercel.app**

### Упатство за наставникот (брз почеток)

1. Отвори ја страницата на прелиставачот
2. Кликни **„Наставник"**
3. Внеси соодветен код за соба (или генерирај автоматски)
4. Сподели го кодот со учениците
5. Почекај учениците да се приклучат (гледаш ги во реално време)
6. Кликни **„Започни игра"**
7. По завршување — прегледај го извештајот по ученик

### Упатство за ученикот

1. Отвори ја страницата на прелиставачот
2. Кликни **„Ученик"**
3. Внеси го кодот за соба (добиен од наставникот)
4. Внеси надимок
5. Чекај наставникот да ја стартува играта
